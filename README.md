# Weather Dashboard

## Описание проекта
Weather Dashboard — это Android-приложение для отображения погодных данных (температура, влажность, скорость ветра и индекс погоды) с имитацией загрузки из разных источников. Приложение демонстрирует использование корутин Kotlin для асинхронных операций без блокировки UI-потока.

## Функциональность
- Загрузка данных о температуре, влажности и скорости ветра с имитацией сетевых задержек
- Параллельная загрузка данных с помощью `async` (время загрузки ~2 секунды вместо 4.5)
- Отображение индикатора загрузки и текстового прогресса
- Вычисление индекса погоды на основе всех параметров (тяжёлые вычисления на `Dispatchers.Default`)
- Кнопка ручного обновления данных
- Кнопка симуляции ошибки для тестирования обработки исключений
- Автоматическое обновление данных каждые 10 секунд
- Отображение ошибок с возможностью повторной загрузки

## Технологии и библиотеки
- Kotlin
- Jetpack Compose (UI)
- ViewModel (архитектурный компонент)
- StateFlow (реактивное состояние)
- Coroutines (асинхронность)
  - `viewModelScope`
  - `launch`, `async`, `await`
  - `Dispatchers.Main`, `Dispatchers.IO`, `Dispatchers.Default`
  - `coroutineScope`, `withContext`
- Material Design 3

## Контрольные вопросы

### A) В чём разница между launch и async?

| Метод | Возвращает | Когда использовать |
|-------|------------|-------------------|
| `launch` | `Job` | Запустить задачу "забыл и выполнил" (логирование, отправка уведомлений) |
| `async` | `Deferred<T>` | Нужен результат выполнения (загрузка данных, вычисления) |

**Пример кода:**
```kotlin
// launch - не ждём результат
viewModelScope.launch {
    sendAnalyticsEvent()
}

// async - ждём результат
val result = viewModelScope.async {
    fetchData()
}.await()
```

### B) Что такое suspend функция?
`suspend` — это функция, которая может приостановить выполнение корутины без блокировки потока.

**Может ли вызываться из обычной функции?**
Нет, `suspend` функцию можно вызывать только из другой `suspend` функции или внутри корутины (`launch`/`async`).

**Почему `delay()` не блокирует поток?**
`delay()` приостанавливает только текущую корутину, а поток освобождается для других задач. В отличие от `Thread.sleep()`, который блокирует весь поток.

### C) Зачем нужны разные диспетчеры?

| Диспетчер | Задачи | Пример |
|-----------|--------|--------|
| `Dispatchers.Main` | Обновление UI | Изменение текста в `Text`, показ `CircularProgressIndicator` |
| `Dispatchers.IO` | Ввод/вывод | Сетевые запросы, работа с базой данных, чтение файлов |
| `Dispatchers.Default` | Тяжёлые вычисления | Сортировка больших массивов, обработка изображений |

**Что будет, если выполнить тяжёлое вычисление на `Dispatchers.Main`?**
UI зависнет (ANR — Application Not Responding), потому что главный поток будет занят вычислениями и не сможет обрабатывать события интерфейса.

### D) Что произойдёт, если не обработать исключение в корутине?
Приложение упадёт с `FATAL EXCEPTION`, как в примере с необработанной ошибкой в `async` до использования `coroutineScope`.

**Как корректно обрабатывать ошибки?**
```kotlin
viewModelScope.launch {
    try {
        coroutineScope {
            val deferred = async { repository.fetchData() }
            val result = deferred.await()
            // обновление состояния
        }
    } catch (e: Exception) {
        // обработка ошибки
        _state.value = _state.value.copy(error = e.message)
    }
}
```

**Зачем нужен `try-catch` внутри `launch`?**
Чтобы перехватить исключения, которые могут возникнуть в дочерних корутинах, и корректно обновить UI (показать сообщение об ошибке, скрыть индикатор загрузки).

### E) Как работает автоматическая отмена корутин?
`viewModelScope` — это область видимости корутин, привязанная к жизненному циклу `ViewModel`. При вызове `onCleared()` (ViewModel уничтожается) все запущенные в этом scope корутины автоматически отменяются.

**Когда корутины отменяются автоматически?**
- При закрытии экрана (Activity/Fragment уничтожается)
- При повороте экрана (ViewModel пересоздаётся)
- При вызове `finish()` или `onBackPressed()`
